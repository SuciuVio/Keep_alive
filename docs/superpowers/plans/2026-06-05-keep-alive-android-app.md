# Keep-Alive Android App Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a native Android app that keeps configured Render/free-tier URLs warm by pinging them every 30 seconds while monitoring is active, and alerts after 3 consecutive failures per URL.

**Architecture:** A Kotlin Android app stores URLs in SharedPreferences through `UrlRepository`, displays them in `MainActivity`, and runs checks in `PingService` as a foreground service. `NotificationHelper` owns both the ongoing service notification and per-URL down-server alert notifications.

**Tech Stack:** Kotlin, Android SDK, Android Gradle Plugin, kotlinx-coroutines-android, Gson, AndroidX RecyclerView.

### Task 2: Storage and Notifications

**Files:**
- Create: `app/src/main/java/com/keepalive/UrlRepository.kt`
- Create: `app/src/main/java/com/keepalive/NotificationHelper.kt`

- [x] **Step 1: Implement URL persistence**

`UrlRepository` reads and writes a JSON string list in SharedPreferences. It normalizes URL input with `trim()`, avoids duplicates, and returns mutable copies.

- [x] **Step 2: Implement notifications**

`NotificationHelper` creates one low-importance foreground channel and one default-importance alert channel. Foreground notification is ongoing; down alert notification opens `MainActivity`.

- [x] **Step 3: Commit**

Run `git add .` and `git commit -m "feat: storage and notification helpers"`.

### Task 3: Ping Foreground Service

**Files:**
- Create: `app/src/main/java/com/keepalive/PingService.kt`

- [x] **Step 1: Implement service lifecycle**

Handle `ACTION_START` and `ACTION_STOP`, use `START_STICKY`, avoid duplicate ping jobs, and cancel the coroutine scope on destroy.

- [x] **Step 2: Implement ping loop**

Every 30 seconds, read the latest URL list, send HTTP GET to each URL, broadcast a timestamped log line, and update the foreground notification URL count.

- [x] **Step 3: Implement down-server alerts**

Track consecutive failures and alert state by URL. Notify after 3 consecutive failures only once per failure episode, reset on success, and remove stale state for deleted URLs.

- [x] **Step 4: Commit**

Run `git add .` and `git commit -m "feat: PingService ForegroundService cu alerte down"`.
